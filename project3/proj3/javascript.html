About JavaScript

JavaScript, JScript, and ECMAScript

  1. JavaScript is a programming language that is frequently utilized in web development. Its code can only be executed on a web browser. In addition to HTML and CSS, JavaScript is a fundamental component of the World Wide Web. Brendan Eich was the creator of JavaScript, which debuted on December 4, 1995.
  2. Since JScript was a variation of Microsoft's JavaScript, it is identical to JavaScript. Because Oracle Corporation is the trademark of JavaScript, Microsoft intended to avoid trademark difficulties when implementing the program, which is why it was dubbed JScript.
  3. ECMAScript is a formal standard (specifically, ECMA-262) developed by Ecma International to ensure that scripting languages like JavaScript behave consistently across different platforms and browsers. The European Computer Manufacturers Association (ECMA) developed the standard, which is now maintained by Ecma International. The first edition of ECMA-262 was adopted by the Ecma General Assembly in June 1997. Several editions of the language standard have been published since then. {14}

Increment Operators

To raise the variable's value by one, use the JavaScript increment(+ +) operator. Where you place the ++ determines the value that the operand returns:

  • prefix increment: When the ++ operator is placed before the variable, the value is increased by one and then returned.
  • postfix increment: When the ++ operator is placed after the variable, the value is returned first, and then incremented.
The increment operator can only be applied on variable and object properties, meaning it can only be used on references. Furthermore, it is not possible to chain the increment operator.

Example:

let i = 5;
console.log(++i); // prints 6
console.log(i++); // prints 6
console.log(i); // prints 7

Function Rest Parameters

Rest parameters enables a function to take a variable amount of arguments. They are indicated by three dots (...) followed by the name of the parameter. All of the arguments supplied to the function can then be accessed using this parameter name.
When working with functions that accept an unknown amount of arguments, rest parameters are quite helpful. Instead of having to iterate through all of the arguments and store them in an array, rest parameters allow you to easily access all of the arguments.

Example:

function sum(...numbers) {
  let total = 0;
  for (let n of numbers) total += n;
  return total;
}
console.log(sum(2, 4, 6, 8)); // prints 20

Swap Function

In JavaScript, when you pass primitive types such as numbers, strings, or booleans to a function, they are passed by value. This means the function receives a separate copy of the variableâ€™s value, not the original one. As a result, any modifications made inside the function do not change the original variable outside of it.
This makes it impossible to create a swap function that exchanges the values of two primitive variables directly. However, you can achieve a similar effect by using other ways such as:

  • Returning Swapped Values: To make the swap effective, you can return the new values and assign them back to the original variables.

    • Example:

    function swap(a, b) {
      return [b, a];
    }
    let [x, y] = swap(10, 20);
    console.log(x, y);

  • Swapping by Reference (Objects or Arrays): JavaScript provides a **reference to the object's memory address** when you supply an object or array. This implies that its contents can be changed within the function. The original reference won't be impacted even if the parameter is reassigned.

    Example:

    function swapProps(obj) {
      let temp = obj.a;
      obj.a = obj.b;
      obj.b = temp;
    }
    let pair = { a: 5, b: 9 };
    swapProps(pair);
    console.log(pair.a, pair.b); // prints 9 5

Regular Expressions

A Regular Expression is a sequence of characters that forms a search pattern. Regex is a common shorthand for a regular expression.
JavaScript RexExp is an Object for handling Regular Expressions. RegExp are be used for:

  • Text searching
  • Text replacing
  • Text validation
Regular Expression Syntax : /pattern/modifier flags;

Regular expression flags are parameters that can modify how a pattern is used, such as making it case-insensitive or global.
These are the most common:
  • g - Global search (find all)
  • i - Case-insensitive search
  • m - Multi-line search
  • u - Enables Unicode support (new 2015)

Example:

Do a case-insensitive search for "ABC" in a string:
let text = "abc defg aBc";
let n = text.search(/ABC/i);

Regular expressions are often used with the string methods:
  • match(regex): returns an Array of results
  • replace(regex): returns a new string
  • search(regex): returns the index of the first match

Example:

Replace "Hello" with "Hi" in a string:
let text = "Hello World";
let result = text.replace(/Hello/i, "Hi");;

Regex in python

Python has a built-in package called re, which can be used to work with Regular Expressions.
Import the re module: import re
When you have imported the re module, you can start using regular expressions:

Example:

Search the string to see if it starts with "The" and ends with "Spain":
import re
txt = "The rain in Spain"
x = re.search("^The.*Spain$", txt)

The re module offers a set of functions that allows us to search a string for a match:
  • findall(): Returns a list containing all matches
  • search(): Returns a Match object if there is a match anywhere in the string
  • split(): Returns a list where the string has been split at each match
  • sub(): Replaces one or many matches with a string
