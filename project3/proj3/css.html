CSS: History and Usage Across Devices

History of CSS

CSS is a stylesheet language used to control the presentation of HTML and XML documents, including SVG and XHTML. It was created in 1994 by the W3C with input from Bert Bos and Håkon Wium Lie. Its primary objective was to decouple presentation from content, increasing the adaptability and flexibility of web development.
While CSS1 (1996) introduced the box model and basic text styling, CSS2 (1998) brought sophisticated selectors, positioning, and media types. Inconsistencies were rectified in CSS 2.1 (2011), and CSS3 introduced modular features such as animations, transitions, flexbox, grid layouts, and responsive design through media queries. With additional selectors, properties, and functions like variables, container queries, and improved layout capabilities, CSS is now a living standard that ensures developers can construct dynamic, responsive, and accessible web designs.{13}

@-Rules in CSS

One of CSS’s powerful innovations is the concept of @-rules. These rules CSS at-rules are statements that instruct CSS on how to behave. Each at-rule begins with an @ followed by a name.
Here are some commonly used at-rules:

  • @import bring one CSS file into another.
  • @media apply styles depending on device features (screen size, resolution, orientation).
  • @font-face load and use custom fonts in the webpage.
  • @keyframes define steps of CSS animations.
  • @supports check if a browser supports a CSS feature before applying it.

CSS across different devices

Making websites that work and look fantastic across a range of screens and devices is essential to responsive web design. For this, CSS media queries are an effective tool. It enables you to apply several styles according to particular parameters like screen size, resolution, or orientation.
Let us discuss how to use CSS to target particular screen sizes or devices;

  1. Media Queries:
    Media queries let CSS adapt a website’s layout and style to different screen sizes, orientations, and dimensions.
    • Apply styles based on screen width, height, or orientation.
    • @media (max-width: 768px) target tablets and mobiles.
    • @media (max-height: 500px) target shorter screens.
    • Combine conditions with and, e.g. portrait orientation + width limit.
  2. Using Device Pixel Ratio:
    Device Pixel Ratio (DPR) allows developers to optimize content for high-resolution screens like Retina displays.
    • Target high-DPI devices using DPR queries.
    • Ensure images and text remain sharp on modern displays.
    • Common practice: provide higher-resolution graphics for devices with DPR ≥ 2.
