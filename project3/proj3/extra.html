How do websites secure your passwords?

When learning about encryption and strong passwords, it's also crucial to understand how passwords are stored on servers. Once data is sent to a server, it's beyond your control, so servers should store minimal information and only in encrypted form. SQL injections and database breaches can expose plain-text data, which is why secure servers use hashing instead of storing passwords directly. Hashing converts a password into a fixed, unreadable string using algorithms like SHA-256 or SHA-512, often combined with PBKDF2 for added protection against brute-force attacks. Modifying $x in a PHP foreach loop

Let's take this loop for example:
$arr = [1, 2, 3, 4, 5];
foreach ($arr as $x) {
$x = $x * 2;
}
After executing this code, the original array $arr remains unchanged as [1, 2, 3, 4, 5]. This is because in a foreach loop, $x is a copy of each element in the array,

On the other hand, if you want to modify the original array elements, you can use a reference by adding an ampersand (&) before $x:
$arr = [1, 2, 3, 4, 5];
foreach ($arr as &$x) {
$x = $x * 2;
}
Here, the output will be [2, 4, 6, 8, 10] because $x is a reference to each element in the array, allowing you to modify the original array directly.
